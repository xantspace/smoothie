Design Section 2 — Animation Sequence

The full dramatic blend flow, phase by phase

Phase 1 · 0 – 1.2s
Ingredient Drop-In
Each selected ingredient falls one-at-a-time from above the blender lid into the jar. On landing, a small ripple/splash ring radiates outward from the liquid surface. The liquid tints slightly with each ingredient's color.
~0.4s per ingredient · staggered · cubic-bezier drop curve
CSS @keyframes SVG ripple circle JS stagger timing
Phase 2 · 1.2 – 3.2s
Full Blend — Blade Frenzy
Blade spins at high speed with a motion-blur CSS filter. Liquid churns — a skew + scaleY oscillation makes it look like it's sloshing violently. Bubbles (small SVG circles) rise from the blade and pop at the surface. The jar gets a subtle vibration shake (translateX ±2px).
2s duration · blade: 0.08s rotate loop · shake: 0.05s loop
CSS blur filter CSS skewX + scaleY JS bubble spawner
Phase 3 · 3.2 – 4.5s
Settle & Color Lock
Blade decelerates (spin animation easing to stop). Churning slows. Liquid surface flattens with a gentle final wave. The liquid transitions to its final blended color (calculated average of all ingredient colors). Power button indicator flips from red to green.
1.3s settle · CSS transition on fill color
CSS ease-out deceleration SVG fill transition
Phase 4 · 4.5 – 6.5s
Pour Into Glass
A glass SVG slides in from the right. The blender jar tilts slightly (CSS rotate). A smooth liquid stream SVG path animates from the jar spout into the glass — using a stroke-dashoffset trick to draw the stream. The glass fills from bottom to top with the blend color.
2s total · glass slide: 0.4s · stream draw: 0.8s · fill rise: 1.2s
CSS translateX slide-in SVG stroke-dashoffset stream SVG clipPath fill rise
Phase 5 · 6.5s+
Result State
The glass stays on screen. A result label fades in below it with the smoothie name (e.g., "Mango & Strawberry Blend"). A "Blend Again" button replaces "Blend It!" and resets the whole sequence — glass slides out, blender resets, ingredients re-enable.
